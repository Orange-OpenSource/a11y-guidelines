Structured Concurrency #


The concurrent execution is based on tasks whose structure define a specific way to handle the situation.

When the number of tasks to be completed is known, the async let syntax can be used to abide by the structure concurrency.

It's highly recommended to use the structured Tasks so as to let Swift doing smart things such as cancellation whose feasibility is due to the parent-child relationship.

The unstructured concurrency relies on tasks that aren't part of any hierarchy and so don't have a parent task.

Priority #

A task priority is defined as communicating to the system how urgent a specific task is.
This priority level inherited from the parents' in the task tree becomes a priority inversion when a high-priority task is waiting on the result of a lower-priority task.

A structured priority escalation comes in when low priority tasks are awaited by a higher priority task to be modified.

Local value #

A task-local value is a variable associated with a task hierarchy and declared as a static property with a dedicated wrapper.
This value exists in a specific scope and gets back to its original value at the end thereof.

All tasks but detached ones inherit task-local values from the current task.

SwiftLog #

It's recommended to utilize the SwiftLog API package when dealing with a server-side Swift application or a cross-platform app/library to use crucial elements for debugging: the loggers to get messages in a certain log level.

The MetadataProvider API is a valuable tool to combine information and merge into a single object presented by a Logger.

Traces #

Besides the Swift Concurrency and the HTTP traffic instruments that are useful for debugging, a new open source package for server ecosystem is introduced: Swift Distributed Tracing

Instrument the code with additional information provided thanks to the withSpan API or its attributes fills the tracing system with appropriate insights into the task hierarchy.

When timing races give rise to errors, the deep connection between tasks and time in the tree provides useful information in the tracing system to find out the source of the problem.
