Stepbar or multi-step process

Introduction

In this example we will see how to improve the vocalization of a “stepbar” component.

To achieve this we will use the stepbar component available in the Boosted library. Some styles have been modified to fit onto a single line.

Non accessible example

Here is the final appearance of the stepbar.

  • 1 Identification
  • 2 Shipping address
  • 3 Delivery options
  • 4 Payment
  • 5 Confirmation

Testing the non-accessible example

Keyboard navigation

For the stepbar we decided that the items shouldn't be clickable. So there is nothing to do.

Screen reader navigation

At this stage, the vocalization of the stepbar is unsatisfactory.

If you listen to the component using a screen reader, a list of 5 items is read. Each item is properly vocalized; however, we have no indication of the current step.

Vocalization support

To improve the vocalization of this component it is necessary to think about the information that seems relevant to the user.

Visually, when we see this stepbar we have access to the following information:

  • The name of the current step,
  • the number of total steps,
  • the number of remaining steps,
  • the name of each step.

Ideally we should have the same level of information with as without a screen reader. That being said, it can be tedious to have too much information to listen to. Therefore, we will focus on the bold information listed above (the three first items in our list).

Possible example of vocalization: “Delivery options, step 3 of 5.”

Technically, there are several ways to achieve this. We could try to hide inactive items from the screen reader and modify the vocalization of the selected item. However, given that this component changes according to the width of the screen (responsive design), the easiest way is to completely hide this component from the screen reader with the aria-hidden="true" attribute. Finally, add an accessible hiding span containing the text to vocalize.

Accessible hiding allows to hide a text, but ensures that it is correctly vocalized by the screen reader. It is done through a simple CSS class. In the Boosted and Bootstrap libraries there is an accessible hiding class named sr-only.

See the accessible hiding example for more information.

Accessible example

Delivery options, step 3 of 5
  • 1 Identification
  • 2 Shipping address
  • 3 Delivery options
  • 4 Payment
  • 5 Confirmation

This new example now vocalizes: “Delivery options, step 3 of 5”.
